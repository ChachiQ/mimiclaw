# 贡献指南

感谢您对 MimiClaw 的关注。本项目欢迎专注、高质量的贡献，致力于提升固件的可靠性、性能或文档质量。

## 贡献范围

我们接受以下方面的贡献：

- 核心固件功能与缺陷修复
- 文档、示例与图表
- 构建和工具链改进
- 测试与 CI 增强

> **外部外设**
> 目前我们暂不接受新增或修改外部外设的 PR。

## 开始之前

- 搜索现有 Issue 和讨论，避免重复工作。
- 对于大规模或高风险的变更，请先提交简短的提案。
- 尽可能保持变更小而可审查。

## 开发环境配置

- 安装 ESP-IDF v5.5+。
- 构建目标通过 `idf.py` 配置。
- 默认配置文件位于 `main/mimi_secrets.h.example`。

## 分支与提交

- 使用简短、描述性的分支名称。
- 保持提交历史整洁且聚焦。
- 建议的提交消息格式：`docs: ...`、`fix: ...`、`feat: ...`。

## Pull Request

- 清晰描述问题及解决方案。
- 包含测试步骤和结果。
- 行为发生变化时同步更新文档。

## 代码风格

- 遵循现有的命名规范和代码格式。
- 以清晰为先，不追求炫技。
- 避免将大规模重构与功能性变更混在同一个 PR 中。

## 测试

- 行为发生变化时，添加或更新测试。
- 如果无法提供测试，请说明原因以及您是如何验证变更的。

## 文档

- 行为发生变化时，保持 README 和文档同步更新。
- 为新功能添加简明的使用示例。

## 许可证

提交贡献即表示您同意，您的贡献将以 MIT 许可证进行授权。
